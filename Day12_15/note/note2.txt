代码规范-CSS
1、【强制】使用4个空格作为一个缩进层级，不允许使用2个空格或tab字符。
2、【强制】选择器与{之间必须包含空格。
3、【强制】属性名与：之间不允许有空格，：与属性值之间必须包含空格。
4、【强制】列表型属性值书写在单行时，后必须跟一个空格。
5、【强制】当一个rule包含多个selector时，每个选择器声明必须独占一行。
6、【强制】>、+、~选择器的两边各保留一个空格。
7、【强制】属性选择器中的值必须用双引号包围，不允许使用单引号，不允许不使用引号。
8、【强制】属性定义后必须以分号结尾。
9、【强制】如无必要，不得为id、class选择器添加类型选择器进行限定。
10、【建议】选择器的嵌套层级应不大于3级，位置靠后的限定条件应尽可能精准。
11、【建议】在可以使用缩写的情况下，尽量使用属性缩写。
12、【建议】使用border/margin/padding等缩写时，应注意隐含值对实际数值的影响，确实需要设置多个方向的位置时才使用缩写。
13、【建议】书写顺序：布局方式、位置 > 尺寸 > 文本相关 > 视觉效果
• Formatting Model 相关属性包括：position / top / right / bottom / left / float / display / overflow 等
• Box Model 相关属性包括：border / margin / padding / width / height 等
• Typographic 相关属性包括：font / line-height / text-align / word-wrap 等
• Visual 相关属性包括：background / color / transition / list-style 等
14、【建议】尽量不使用！important声明，当需要强制指定样式且不允许任何场景覆盖时，通过标签内联和！important定义样式。
15、【建议】将z-index进行分层，对文档流外绝对定位元素的视觉层级关系进行管理；在可控环境下，期望显示在最上层的元素，z-index指定为999999；在第三方环境下，期望显示在最上层的元素，通过标签内联和 !important，将 z-index 指定为 2147483647。
16、【强制】文本内容必须用双引号包围。
17、【强制】当数值为0-1之间的小数时，省略整数部分的0。
18、【强制】url()函数中的路径不加引号。
19、【建议】url()函数中的绝对路径可省去协议名。
20、【强制】长度为0时须省略单位（也只有长度单位可省）。
21、【强制】RGB颜色值必须使用十六进制记号形式#rrggbb.不允许是用rgb()
解释：带有alpha的颜色信息可以使用rgba()。使用rgba()时每个逗号后必须保留一个空格。
22、【强制】颜色值可以缩写时，必须使用缩写形式；颜色值不允许使用命名色值。
23、【强制】2D位置：必须同时给出水平和垂直方向的位置。
24、【强制】font-family 按「西文字体在前、中文字体在后」、「效果佳 (质量高/更能满足需求) 的字体在前、效果一般的字体在后」的顺序编写，最后必须指定一个通用字体族( serif / sans-serif )。
25、【强制】font-family不区分大小写，但在同一个项目中，同样的Family Name大小写必须统一。
26、【强制】需要在windows平台显示的中文内容，其字号应不小于12px。
27、【强制】font-weight属性必须使用数值方式描述。
28、【强制】使用transition时应指定transition-property。
29、【强制】Media Query不得单独编排，必须与相关的规则一起定义；如果有多个逗号分隔的条件时，应将每个条件放在单独一行中。
30、【强制】带私有前缀的属性由长到短排列，按冒号位置对齐。
